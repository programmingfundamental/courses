# Упражнение: Създаване на изпълним файл за JavaFX приложение с Maven и jpackage

# Какво е пакетиране и изпълним файл

## Какво е пакетиране на софтуерно приложение

Пакетирането (packaging) представлява процесът на подготовка на едно приложение за разпространение и стартиране извън средата за разработка (IDE).

По време на разработката програмата се състои от:

* изходен код (`.java`);
* външни библиотеки;
* ресурсни файлове;
* конфигурации;
* зависимости.

Тези файлове не могат директно да бъдат използвани от крайния потребител.
Поради тази причина е необходимо приложението да бъде пакетирано в завършен и удобен за стартиране вид.

---

# Основни цели на пакетирането

Пакетирането има няколко основни цели:

## 1. Обединяване на файловете

Всички необходими компоненти на приложението се събират в единна структура:

* компилиран код;
* библиотеки;
* ресурси;
* конфигурационни файлове;
* runtime зависимости.

---

## 2. Подготовка за разпространение

След пакетиране приложението може да бъде:

* копирано на друг компютър;
* инсталирано;
* стартирано от потребители без IDE.

---

## 3. Осигуряване на изпълнима среда

При Java приложения често е необходимо да се добави и Java Runtime Environment (JRE) или custom runtime image, за да може приложението да работи независимо от инсталираната Java версия.

---

# Какво е изпълним файл

Изпълним файл (Executable File) е файл, който съдържа инструкции, които операционната система може директно да стартира.

Различните операционни системи използват различни формати:

| Операционна система | Изпълним файл |
| ------------------- | ------------- |
| Windows             | `.exe`        |
| Linux               | бинарен файл  |
| macOS               | `.app`        |

---

# Изпълним файл при Java приложения

При Java приложенията ситуацията е по-специфична, защото Java кодът не се изпълнява директно от операционната система.

Процесът е следният:

```text
.java → .class → JVM
```

## Етапи

1. Java кодът се компилира до `.class` файлове;
2. `.class` файловете се изпълняват от Java Virtual Machine (JVM);
3. За да се улесни стартирането, приложението може да бъде пакетирано като:

   * `.jar` файл;
   * executable application image;
   * installer.

---

# Какво е JAR файл

JAR (Java ARchive) е архивен файл, който съдържа:

* компилирани `.class` файлове;
* ресурси;
* метаинформация.

Пример:

```text
my-app.jar
```

JAR файлът обаче изисква:

* инсталирана Java;
* правилна версия на Java;
* ръчно стартиране чрез команда.

Пример:

```bash
java -jar my-app.jar
```

---

# Какво е executable application image

Executable application image е пакетирано приложение, което съдържа:

* изпълним файл;
* необходимите библиотеки;
* Java runtime;
* конфигурации.

То може да бъде стартирано директно чрез двоен клик.

Пример:

```text
DeveloperApp.exe
```

---

# Разлика между JAR файл и изпълним файл

| Характеристика                 | JAR файл     | Изпълним файл |
| ------------------------------ | ------------ | ------------- |
| Изисква Java                   | Да           | Обикновено не |
| Стартиране                     | Чрез команда | С двоен клик  |
| Удобство за потребителя        | По-ниско     | По-високо     |
| Съдържа runtime                | Не           | Да            |
| Подходящ за крайни потребители | Ограничено   | Да            |

---

# Пакетиране при JavaFX приложения

JavaFX приложенията използват допълнителни библиотеки:

* `javafx-controls`
* `javafx-fxml`
* `javafx-graphics`

След Java 11 JavaFX вече не е част от стандартния JDK.
Затова при пакетиране трябва:

1. JavaFX модулите да бъдат включени;
2. да се зададе module path;
3. да се посочат необходимите модули.

---

# Роля на jpackage

`jpackage` е инструмент, който:

* създава executable application image;
* генерира installer;
* добавя Java runtime;
* подготвя приложението за крайни потребители.

---

# Примерен процес на пакетиране

## 1. Компилиране

```text
.java → .class
```

---

## 2. Създаване на JAR

```text
.class → application.jar
```

---

## 3. Добавяне на зависимости

```text
JavaFX + библиотеки + runtime
```

---

## 4. Създаване на executable image

```text
DeveloperApp.exe
```

---

Пакетирането е важен етап от разработката на софтуер, чрез който приложението се подготвя за разпространение и използване от крайни потребители.

Изпълнимият файл представлява завършен вариант на приложението, който може директно да бъде стартиран от операционната система без необходимост от среда за разработка.

При JavaFX приложенията пакетиране се използва не само за удобство, но и за осигуряване на необходимите JavaFX библиотеки и Java runtime среда.
