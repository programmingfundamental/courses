JavaFX използва CSS (Cascading Style Sheets) за отделяне на визуалното оформление (цветове, шрифтове, разстояния) от структурата и логиката на приложението, подобно на уеб разработката. JavaFX CSS се базира на спецификацията CSS 2.1.

### 1. Специфики на JavaFX CSS

- **Префикс `-fx-`:** Всички специфични за JavaFX свойства задължително започват с префикса `-fx-` (напр. `-fx-background-color` вместо `background-color`).
- **Default Style:** По подразбиране JavaFX приложенията използват вградената системна тема, наречена _Modena_.
- **Включване в FXML:** За да приложите външен CSS файл към вашия изглед, добавете атрибута `stylesheets` към кореновия (root) елемент (подробно обяснено в следващата точка):
  ```xml
  <VBox stylesheets="@styles.css" xmlns:fx="http://javafx.com/fxml/1">
      ...
  </VBox>
  ```

### 2. Как се подава пътят към CSS файла (Атрибут `stylesheets`)

При указването на пътя в JavaFX има няколко специфични правила, които зависят от структурата на проекта (ресурсните файлове се намират в папка `src/main/resources`).

#### 2.1. Символът `@` (Указател за ресурс)

В FXML пътищата до външни файлове (като CSS или изображения) **задължително** трябва да започват със символа `@`. Той казва на FXMLLoader, че следващият текст не е просто обикновен низ, а път към файл (ресурс), който трябва да бъде намерен в проекта.

#### 2.2. Относителен път (Relative Path)

Относителният път търси CSS файла спрямо местоположението на самия FXML файл.

- **Когато FXML и CSS файлът са в една и съща папка:**

  ```xml
  <VBox stylesheets="@styles.css" xmlns:fx="http://javafx.com/fxml/1">
      <!-- Съдържание -->
  </VBox>
  ```

- **Когато CSS файлът е в подпапка спрямо FXML файла:**

  Ако вашият FXML е в папка `views`, а CSS файлът е в `views/css/`, пътят изглежда така:

  ```xml
  <VBox stylesheets="@css/styles.css" xmlns:fx="http://javafx.com/fxml/1">
      <!-- Съдържание -->
  </VBox>
  ```

#### 2.3. Абсолютен път (Absolute Path)

Абсолютният път (спрямо корена на ресурсите) е най-сигурният и често препоръчван метод при по-големи проекти. Той се указва чрез добавяне на наклонена черта `/` веднага след `@`.

Знакът `/` представлява началната папка за ресурси на проекта (т.е. `src/main/resources`). По този начин няма значение къде точно се намира FXML файлът – пътят винаги ще се чете от главната папка с ресурси.

- **Пример:** Ако вашият CSS файл се намира точно в `src/main/resources/css/main.css`, пътят винаги ще бъде:
  ```xml
  <VBox stylesheets="@/css/main.css" xmlns:fx="http://javafx.com/fxml/1">
      <!-- Съдържание -->
  </VBox>
  ```

#### 2.4. Добавяне на множество CSS файлове

Ако вашият изглед има нужда от повече от един CSS файл, можете да ги изброите в атрибута `stylesheets`, като ги разделите със запетая:

```xml
<VBox stylesheets="@/css/normalize.css, @/css/main.css">
    <!-- Съдържание -->
</VBox>
```

### 3. Селектори

Селекторите определят към кои графични елементи ще бъде приложен даден стил:

1. **Типов селектор:** Прилага се за всички елементи от даден тип. В CSS името на класа се изписва с малки букви.
   ```css
   .button {
     -fx-background-color: #e0e0e0;
   }
   ```
2. **Стилов клас (Style Class):** Позволява прилагане на стил към конкретна група елементи. В CSS се използва символът `.` (точка). В FXML се задава чрез атрибута `styleClass`.
   - _FXML:_

   ```xml
   <Label text="Грешка!" styleClass="error-text" />
   ```

   - _CSS:_
     ```css
     .error-text {
       -fx-text-fill: red;
     }
     ```

3. **ID селектор:** Прилага се за един конкретен елемент. В CSS се използва символът `#`. В FXML се задава чрез атрибута `id` (забележка: това е различно от `fx:id`, въпреки че често имат еднакви стойности).
   - _FXML:_

   ```xml
   <Button text="Вход" id="login-button" />
   ```

   - _CSS:_
     ```css
     #login-button {
       -fx-font-weight: bold;
     }
     ```

### 4. Списък с най-често използвани стилове

#### Фон и запълване (Backgrounds & Fills)

- `-fx-background-color`: Задава цвета на фона (имена, Hex, RGB или градиенти).
- `-fx-background-radius`: Определя заоблянето на ъглите.
- `-fx-background-insets`: Отстояние на фона от границите на елемента.

```css
.my-pane {
  -fx-background-color: linear-gradient(to bottom, #ffffff, #e0e0e0);
  -fx-background-radius: 10;
}
```

#### Граници и рамки (Borders)

- `-fx-border-color`: Цвят на рамката.
- `-fx-border-width`: Дебелина (напр. `2` за всички страни или `2 0 2 2` за Горе, Дясно, Долу, Ляво).
- `-fx-border-radius`: Заобляне на рамката.

```css
.custom-textfield {
  -fx-border-color: #cccccc;
  -fx-border-width: 0 0 2 0; /* Само долна рамка */
}
```

#### Текст и шрифтове (Text & Fonts)

- `-fx-text-fill`: Задава цвета на текста в контроли като бутони и етикети.
- `-fx-font-family`: Избор на шрифт (напр. `'Arial'`).
- `-fx-font-size`: Размер (напр. `14px` или `1.2em`).
- `-fx-font-weight`: Дебелина (`bold`, `normal`).
- `-fx-alignment`: Подравняване на съдържанието (`center`, `center-left`).

```css
.header-label {
  -fx-text-fill: #333333;
  -fx-font-family: 'Verdana';
  -fx-font-size: 18px;
  -fx-font-weight: bold;
}
```

#### Размери, отстояния и ефекти

- `-fx-padding`: Вътрешно отстояние (напр. `10 20` за Горен/Долен 10px, Ляв/Десен 20px).
- `-fx-opacity`: Прозрачност на елемента (от `0.0` до `1.0`).
- `-fx-effect`: Добавя графични ефекти като сянка (`dropshadow` или `inner-shadow`).
- `-fx-cursor`: Променя вида на курсора (`hand`, `default`, `wait`).

```css
.card-container {
  -fx-padding: 15;
  -fx-effect: dropshadow(gaussian, rgba(0, 0, 0, 0.2), 10, 0, 0, 4);
}
```

### 5. Псевдо-класове (Състояния)

Псевдо-класовете се използват за динамична промяна на стила при интеракция от страна на потребителя. Добавят се към селектора с двоеточие `:`.

- `:hover` - когато мишката е позиционирана върху елемента.
- `:pressed` - докато елементът е натиснат (кликнат).
- `:focused` - когато елементът е на фокус (напр. мигащ курсор в текстово поле).
- `:disabled` - когато контролата е деактивирана (`disable="true"` в FXML).

**Пример за пълен жизнен цикъл на бутон:**

```css
/* Стандартно състояние */
.action-button {
  -fx-background-color: #2196f3;
  -fx-text-fill: white;
  -fx-background-radius: 5;
  -fx-cursor: hand;
}

/* Когато мишката е отгоре */
.action-button:hover {
  -fx-background-color: #42a5f5;
}

/* При натискане (клик) */
.action-button:pressed {
  -fx-background-color: #1976d2;
}

/* При деактивиран бутон */
.action-button:disabled {
  -fx-background-color: #bdbdbd;
  -fx-text-fill: #757575;
  -fx-cursor: default;
}
```

### 6. Инлайн стилове (Inline Styles)

Ако желаете да приложите специфичен стил само за един елемент и не искате да създавате външен CSS файл, можете да използвате атрибута `style` директно в FXML файла (това отговаря на извикването на метода `setStyle()` в Java). **_Тези стилове имат най-висок приоритет._**

**Пример:**

```xml
<Label text="Успешен запис!" style="-fx-text-fill: green; -fx-font-weight: bold;" />
```

_(Забележка: За големи проекти се препоръчва използването на външни CSS файлове за по-лесна поддръжка)._
