# Лабораторно упражнение 2

### SOLID принципи

SOLID е акроним, който обединява пет основни принципа за обектно-ориентиран дизайн. Те се използват при създаване на софтуер, който е по-лесен за разбиране, разширяване, тестване и поддръжка.

Принципите SOLID не са конкретни правила за синтаксис, а насоки за организиране на класове, интерфейси и зависимости между тях.

| Буква    | Принцип   | Наименование                     |
|----------|-----------|----------------------------------|
| S        | SRP       | Single Responsibility Principle  |
| O        | OCP       | Open/Closed Principle            |
| L        | LSP       | Liskov Substitution Principle    |
| I        | ISP       | Interface Segregation Principle  |
| D        | DIP       | Dependency Inversion Principle   |
