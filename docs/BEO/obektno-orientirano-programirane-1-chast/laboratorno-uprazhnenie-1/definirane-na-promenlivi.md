# Дефиниране на променливи

Декларирането на променливи се прави със следния синтаксис:


\<тип данни\> \<идентификатор\> = \<литерал\>


Присвояването на стойност не е задължително, защото в java можете първо да декларирате променливата и по-късно да присвоите стойността към нея.


Например: Нека променлива с идентификатор num от тип данни int бъде дефинирана. Това означава да има следната дефиниция:


```
int num;
```


Присвояването на стойности на променливите при деклариране изглежда по следния начин:


```
char ch = 'A';
int number = 100;
```


Алтернативна реализация с първоначално дефиниране на променлива и последващо присвояване на стойност:


```
char ch;
int number;
...
ch = 'A';
number  = 100;
```

# Дефиниране на масиви от данни

Декларирането на масив се прави със следния синтаксис:

\<тип данни\>[] \<идентификатор\> [ = { \<литерал\> [,\<литерал\>] } ]

Присвояването на стойност не е задължително, защото в java е възможно първо да бъде декларирана променлива и по-късно да бъде присвоена стойност към нея.

Например: Дефиниране на масив с идентификатор num от тип данни int:

```
int[] num;
```

Присвояването на стойности на променливите при деклариране изглежда по следния начин:

```
char[] ch = { 'I', 'a', 'm' };
int[] number = { 100, 99 };
```

### Конвенция за именуване на променливи в java


* Имената на променливи не могат да съдържат празни пространства, например: int num ber = 100; е невалидно, защото в името на променливата има празно място.
* Според стандартите за кодиране на Java името на променлива трябва да започне с малка латинска буква, например int number;. За променлива, която има повече от една дума в името си, се спазва правилото "първата дума започва с малка буква, всяка следваща дума започва с главна буква": int smallNumber; int bigNumber;
* Идентификаторите на променливите в Java са чувствителни към малки и големи букви букви.\
