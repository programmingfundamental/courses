# Наследяване (Inheritance)

**Наследяването** е фундаментален механизъм в обектно-ориентираното програмиране, който позволява на един клас да приеме характеристиките (атрибути и методи) на друг, вече съществуващ клас. Това помага за изграждането на йерархични структури и значително подобрява преизползваемостта на кода.

### Основни понятия:
* **Суперклас (Superclass / Parent class):** Класът, от който се наследява. Той дефинира общите свойства и поведение.
* **Подклас (Subclass / Child class):** Класът, който наследява. Той придобива функционалността на родителя си и може да добавя свои специфични характеристики или да променя съществуващите.

В Java наследяването се реализира чрез ключовата дума **`extends`**.

---

## Правила и особености в Java

1. **Единично наследяване:** В Java един клас може да наследява директно само **един** родителски клас. Множественото наследяване на класове не е позволено с цел избягване на сложност и конфликти.
2. **Достъпност:** Подкласът наследява всички членове на родителския клас, които не са декларирани като `private`. Обикновено се използва модификаторът `protected`, за да се позволи достъп само в рамките на йерархията.
3. **Еднопосочност:** Родителският клас няма достъп до специфичните методи и полета на своите наследници.

---

## Пример за йерархия при наследяване

Да разгледаме йерархия от класове, описващи животни:

```java
// Базов клас (Родител)
class Animal { 
    void eat() {
        System.out.println("Животното се храни...");
    } 
} 

// Клас наследник на Animal
class Dog extends Animal { 
    void bark() {
        System.out.println("Кучето лае!");
    } 
}

// Клас наследник на Dog (Многостепенно наследяване)
class BabyDog extends Dog { 
    void weep() {
        System.out.println("Малкото куче скимти...");
    } 
}
```

### Използване на наследените методи:

```java
public class Main { 
    public static void main(String[] args) { 
        BabyDog d = new BabyDog(); 
        
        d.weep(); // Метод от BabyDog
        d.bark(); // Наследен метод от Dog
        d.eat();  // Наследен метод от Animal
    }
} 
```

---

## Защо използваме наследяване?

* **Преизползваемост на кода:** Не е необходимо да дефинираме едни и същи методи (например `eat()`) във всеки нов клас.
* **Методи за пренаписване (Method Overriding):** Подкласът може да предостави специфична имплементация на метод, който вече съществува в родителския клас.
* **Полиморфизъм:** Позволява ни да третираме обекти от различни подкласове като обекти от техния общ родителски тип.
