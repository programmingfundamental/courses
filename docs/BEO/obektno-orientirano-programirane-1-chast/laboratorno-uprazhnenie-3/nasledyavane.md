# Декларация на клас

Класът се дефинира чрез ключовата дума class, следвана от неговото име и тяло, оградено с фигурни скоби.

```java
class Student {

}
```

Декларацията на един клас включва следните основни елементи:

* ключовата дума class, която указва началото на дефиницията;
* име на класа, избрано съгласно приетите конвенции за именуване;
* тяло на класа, оградено с фигурни скоби {}, в което се описват неговите членове.

Съгласно конвенциите в Java имената на класовете започват с главна буква и се записват в стил **PascalCase**.

В примера е деклариран клас с име Student. Към момента той не съдържа полета, конструктори или методи и представлява минимална декларация на клас.


# Структура на клас

Класът представлява съвкупност от данни и функционалност, които описват определен тип обекти. Неговата структура се състои от елементи, наречени членове на класа.

В най-общия случай един клас може да съдържа:

* полета (атрибути) – описват състоянието на обектите;
* конструктори – използват се за инициализиране на новосъздадените обекти;
* методи – реализират поведението на обектите.

Методите могат условно да бъдат разделени на:
* аксесори (getter методи) – използват се за извличане на стойността на дадено поле;
* мутатори (setter методи) – използват се за промяна на стойността на дадено поле;
* други методи, реализиращи поведението на обекта.

Пример:
```java
class Student {

    String name;

    String getName() {
        return name;
    }

    void setName(String name) {
        this.name = name;
    }
}
```

В зависимост от конкретната реализация един клас може да съдържа и други елементи, като вложени класове, изброени типове (enum), записи (record), статични блокове и др., които ще бъдат разгледани по-нататък.

Пример:

```java
class Student {

    // Полета
    String name;
    int facultyNumber;

    // Конструктор
    Student(String studentName, int studentFacultyNumber) {
        name = studentName;
        facultyNumber = studentFacultyNumber;
    }

    // Метод
    String fetchStudentInformation() {
         return name + ", " + facultyNumber;
    }
}
```
В примера класът Student съдържа две полета, един конструктор и един метод.

**Полета (атрибути)**

Полетата представляват променливи, декларирани в тялото на класа. Те описват състоянието на обектите и съхраняват информацията, необходима за тяхното представяне.

Всяка инстанция на класа притежава собствен набор от стойности на своите полета.

**Конструктори**

Конструкторът е специален член на класа, който се изпълнява при създаването на нов обект. Основното му предназначение е да инициализира полетата на обекта.

Подробно разглеждане на конструкторите ще бъде направено в следващата точка.

**Методи**

Методите представляват именувани последователности от инструкции, които реализират поведението на обектите. Чрез тях се извършва обработка на данните, съхранявани в полетата на класа.

Методите могат да приемат параметри и при необходимост да връщат резултат.

### Конструктори

Конструкторът е специален член на класа, който се изпълнява при създаването на нов обект. Неговото основно предназначение е да инициализира началното състояние на обекта.

Конструкторът се извиква автоматично при създаването на обект и не може да бъде извикан като обикновен метод.

**Основни характеристики**

Конструкторът притежава следните характеристики:

* името му съвпада с името на класа;
* няма тип на връщаната стойност (дори void);
* може да приема параметри;
* един клас може да съдържа повече от един конструктор.

**Конструктор по подразбиране**

Ако в класа не е дефиниран нито един конструктор, компилаторът автоматично създава конструктор по подразбиране, който не приема параметри.

Пример:

```java
class Student {

    // Полета
    String name;
    int facultyNumber;

}
```
В този случай компилаторът автоматично създава конструктор без параметри.

**Параметризиран конструктор**

Конструкторът може да приема параметри, чрез които при създаването на обекта се задават началните стойности на неговите полета.

Пример:

```java
class Student {

    String name;
    int facultyNumber;

    Student(String studentName, int studentFacultyNumber) {
        name = studentName;
        facultyNumber = studentFacultyNumber;
    }
}
```
В примера стойностите на полетата се задават още при създаването на обекта.

**Предефиниране на конструктор по подразбиране**

При дефиниране на собствен конструктор компилаторът не създава автоматично конструктор по подразбиране.

Ако е необходим конструктор без параметри, той трябва да бъде дефиниран изрично.

Пример:

```java
class Student {

    String name;
    int facultyNumber;

    Student() {}

    Student(String studentName, int studentFacultyNumber) {
        name = studentName;
        facultyNumber = studentFacultyNumber;
    }
}
```

### Ключова дума new

Ключовата дума *new* се използва за създаване на нови обекти в Java. При нейното използване се заделя памет за обекта, извиква се избраният конструктор и се връща референция към новосъздадения обект.

Обектите се създават по време на изпълнение на програмата (runtime).

**Основни действия при използване на new**
При изпълнение на израз с ключовата дума new се извършват следните действия:

* заделя се памет за нов обект;
* извиква се съответният конструктор;
* инициализират се полетата на обекта;
* връща се референция към новосъздадения обект.

Синтаксис:

```java
ClassName objectName = new ClassName();

// или

// ClassName objectName = new ClassName(аргументи);
```

Пример:

```java
class Student {

    String name;
    int facultyNumber;

    Student(String studentName, int studentFacultyNumber) {
        name = studentName;
        facultyNumber = studentFacultyNumber;
    }
}

public class Application {

    public static void main(String[] args) {

        Student student =
                new Student("Ivan Petrov", 12345);

    }
}
```
В примера чрез ключовата дума *new* се създава нов обект от класа Student. Извиква се параметризираният конструктор, а върнатата референция се присвоява на променливата student.

*Забележка:* Ключовата дума new се използва за създаване на обекти от потребителски класове, както и на обекти от класовете на стандартната библиотека на Java.

При използване на ключовата дума new конструкторът работи с новосъздадения обект. Достъпът до този обект от самия конструктор се осъществява чрез ключовата дума this, която ще бъде разгледана в следващата точка.

### Ключова дума this

Ключовата дума *this* представлява референция към текущия обект. Тя се използва за достъп до неговите полета и методи, както и за извикване на друг конструктор от същия клас.

Най-често *this* се използва, когато параметрите на конструктора или метода имат същите имена като полетата на класа.

**Достъп до полетата на текущия обект**

При съвпадение между имената на параметрите и полетата на класа се използва this, за да се направи еднозначно разграничение между тях.

Пример:

```java
class Student {

    String name;
    int facultyNumber;

    Student(String name, int facultyNumber) {
        this.name = name;
        this.facultyNumber = facultyNumber;
    }
}
```
В примера:
* this.name обозначава полето на текущия обект;
* name обозначава параметъра на конструктора.

**Извикване в друг конструктор**

Ключовата дума this може да се използва и за извикване на друг конструктор от същия клас.

Извикването трябва да бъде първата инструкция в конструктора.

```java
class Student {

    String name;
    int facultyNumber;

    Student() {
        this("Unknown", 0);
    }

    Student(String name, int facultyNumber) {
        this.name = name;
        this.facultyNumber = facultyNumber;
    }
}
```
В примера конструкторът без параметри използва параметризирания конструктор, като по този начин се избягва дублирането на код.

**Използване на this при извикване на методи**

Ключовата дума this може да се използва и за извикване на методи на текущия обект.

Пример:

```java
class Student {

    String name;

    void printName() {
        this.printMessage();
    }

    void printMessage() {
        System.out.println(name);
    }
}
```
Използването на this при извикване на методи не е задължително, но често подобрява четимостта на кода.

**Особености**
* this може да се използва само в нестатичен контекст.
* this() трябва да бъде първата инструкция в конструктора.
* this не може да се използва в статични методи и статични блокове.


### Ключова дума static

Ключовата дума static се използва за деклариране на членове, които принадлежат на класа, а не на неговите обекти.

Статичните членове съществуват в единствен екземпляр независимо от броя на създадените обекти и могат да бъдат използвани без създаване на инстанция на класа.

Ключовата дума static може да бъде приложена към:
* полета;
* методи;
* блокове за инициализация.

**Статични полета**

Статичното поле принадлежи на класа и се споделя от всички негови обекти.

При промяна на стойността му промяната е видима за всички инстанции на класа.

Пример:
```java
class Student {

    static String university = "Technical University of Varna";

    String name;

    Student(String name) {
        this.name = name;
    }
}
```
Стойността на полето university е обща за всички обекти от класа Student.

**Достъп до статични полета**

Статичните полета могат да бъдат достъпвани чрез името на класа:

```java
System.out.println(Student.university);
```
Макар достъпът чрез обект да е възможен, препоръчително е статичните членове да се използват чрез името на класа, тъй като принадлежат на него.

**Статични методи**

Статичният метод принадлежи на класа и може да бъде извикан без създаване на обект.

Пример:
```java
class Calculator {

    static int square(int number) {
        return number * number;
    }
}
```

Извикване:
```java
int result = Calculator.square(5);
```

**Особености на статичните методи**

Статичните методи:
* могат директно да използват само статични полета и статични методи;
* не могат да използват ключовата дума this;
* не могат да използват ключовата дума super.

Причината е, че статичните методи принадлежат на класа и не работят с конкретен обект.

**Статични блокове**

Статичният блок представлява специален блок инструкции, който се изпълнява еднократно при зареждането на класа в паметта.

Той се използва за инициализиране на статични полета или за извършване на действия, които трябва да бъдат изпълнени само веднъж.

Синтаксис:
```java
static {

}
```

Пример:

```java
class Student {

    static String university;

    static {
        university = "Technical University of Varna";
        System.out.println("Класът Student е зареден.");
    }
}
```
Статичният блок се изпълнява само веднъж, независимо колко обекта от класа бъдат създадени.

**Обобщение**

| Елемент         | Предназначение                                                    |
|-----------------|-------------------------------------------------------------------|
| статично поле   | Съхранява информация, обща за всички обекти на класа              |
| статичен метод  | Принадлежи на класа и може да бъде извикан без създаване на обект |
| статичен блок   | Изпълнява се еднократно при зареждането на класа                  |

Статични методи се използват широко и в стандартната библиотека на Java. Примери за такива методи са Math.sqrt(), Math.max(), Integer.parseInt() и Double.parseDouble().

### Ключова дума final

Ключовата дума final се използва за ограничаване на възможността за промяна или наследяване. В зависимост от мястото, на което е приложена, тя може да бъде използвана при деклариране на променливи, методи и класове.

**Final променливи**

След инициализиране стойността на променлива, декларирана с ключовата дума final, не може да бъде променяна.

Пример:

```java
final double PI = 3.141592653589793;

System.out.println(PI);
```
Следният код ще доведе до грешка при компилация:

```java
PI = 3.14;
```

**Константи**

Променливите, които представят неизменими стойности през цялото време на изпълнение на програмата, обикновено се декларират като static final.

Съгласно общоприетите конвенции техните имена се записват с главни букви, като отделните думи се разделят със символа _.

Пример:
```java
class MathConstants {

    static final double PI = 3.141592653589793;
    static final int MAX_NUMBER_OF_STUDENTS = 30;

}
```

**Final класове**

Клас, деклариран с ключовата дума final, не може да бъде наследяван.

Пример:
```java
final class Utility {

}
```
Всеки опит за създаване на клас наследник ще доведе до грешка при компилация.

**Обобщение**

| Употреба         | Предназначение                                                |
|------------------|---------------------------------------------------------------|
| final променлива | Стойността не може да бъде променяна след инициализация       |
| final метод      | Методът не може да бъде предефиниран в наследник              |
| final клас       | Класът не може да бъде наследяван                             |



# Наследяване (Inheritance)

**Наследяването** е един от основните механизми на обектно-ориентираното програмиране. То позволява създаването на нов клас въз основа на вече съществуващ клас, като новият клас наследява неговите полета и методи.

По този начин може да бъде изградена йерархия от класове, при която общите характеристики се описват само веднъж в родителския клас, а специализираните – в неговите наследници.

В Java наследяването се реализира чрез ключовата дума *extends*.

### Основни понятия:
При наследяването участват два вида класове:
* **родителски клас** (superclass, parent class) – класът, чиито членове се наследяват;
* **клас наследник** (subclass, child class) – класът, който наследява характеристиките на родителския клас и при необходимост добавя нови полета и методи.

Синтаксис:
```java
class ChildClass extends ParentClass {

}
```

Пример:
```java
class Person {

    String name;

    void printName() {
        System.out.println(name);
    }
}

class Student extends Person {

    int facultyNumber;

}
```
Класът Student наследява полето name и метода printName() от класа Person, като едновременно с това добавя собствено поле facultyNumber.

**Многостепенно наследяване**

В Java е възможно изграждането на йерархия от няколко нива:
```java
class Person {

}

class Student extends Person {

}

class GraduateStudent extends Student {

}
```
Класът GraduateStudent наследява членовете както на Student, така и на Person.

## Правила и особености в Java

Наследяването в Java притежава следните характеристики:
* един клас може да наследява непосредствено само един родителски клас;
* класът наследник получава достъп до наследимите членове на родителския клас;
* наследникът може да добавя нови полета и методи;
* наследникът може да променя поведението на наследени методи. Това ще бъде разгледано по-подробно при изучаването на полиморфизма.


## Предимства

Използването на наследяване предоставя следните предимства:
* повторно използване на вече реализиран код;
* изграждане на логически йерархии между класовете;
* по-лесно разширяване на съществуващи класове;
* намаляване на дублирането на код.
