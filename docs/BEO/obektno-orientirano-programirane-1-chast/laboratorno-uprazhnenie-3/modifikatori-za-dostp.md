# Модификатори за достъп (Access Modifiers)

Модификаторите за достъп определят видимостта на класовете, техните полета, конструктори и методи. Чрез тях се указва откъде могат да бъдат използвани отделните елементи на програмата.

Използването на подходящ модификатор за достъп е основен механизъм за реализиране на капсулацията и за ограничаване на достъпа до вътрешната реализация на класовете.

## Видове модификатори за достъп

В Java съществуват четири нива на достъп:

## 1. `public` (Публичен)

Елементът е достъпен от всяка част на програмата.

Използва се за елементи, които представляват публичния интерфейс на класа.

## 2. `protected` (Защитен)

Елементът е достъпен:

- в рамките на същия пакет;
- в класовете наследници.

## 3. `default` (Достъп на ниво пакет)

Когато не е указан модификатор за достъп, се използва т.нар. достъп на ниво пакет (package-private).

Елементът е достъпен само от класове, намиращи се в същия пакет.

## 4. `private` (Частен)
Елементът е достъпен единствено в рамките на класа, в който е деклариран.

Използва се основно за защита на вътрешното състояние на обекта.

## Обобщаваща таблица на видимостта

| Модификатор     | Клас | Пакет | Подклас | Проект (Външен пакет) |
| --------------- | ---- | ----- | ------- | --------------------- |
| **`public`**    | ✅    | ✅     | ✅       | ✅                     |
| **`protected`** | ✅    | ✅     | ✅       | ❌                     |
| **`default`**   | ✅    | ✅     | ❌       | ❌                     |
| **`private`**   | ✅    | ❌     | ❌       | ❌                     |

## Видове класове

Класът е основният механизъм за дефиниране на потребителски типове данни в Java. Освен обикновените класове езикът предоставя и специализирани видове класове, предназначени за решаване на конкретни задачи.

Най-често използваните видове класове са:

- обикновен клас (Class);
- абстрактен клас (Abstract Class);
- окончателен клас (Final Class);
- изброен тип (enum);
- запис (record).

Следващите точки разглеждат по-подробно специализираните видове класове enum и record.

## Изброени типове (enum)

Изброеният тип (enum) представлява специален вид клас, който се използва за дефиниране на фиксиран набор от именувани константи.

Използването на enum повишава четимостта на програмния код и предотвратява използването на невалидни стойности.

Пример:

```java
enum Status {

    NEW,
    IN_PROGRESS,
    COMPLETED,
    CANCELLED

}
```

Всеки елемент на изброения тип представлява константа.

## Използване

```java
Status status = Status.NEW;
```

или

```java
if (status == Status.COMPLETED) {

}
```

## Вградени методи

| Метод           | Предназначение                       |
| --------------- | ------------------------------------ |
| values()        | Връща масив с всички константи.      |
| valueOf(String) | Връща константа по име.              |
| ordinal()       | Връща поредния номер на константата. |
| name()          | Връща името на константата.          |

Пример:

```java
for (Status status : Status.values()) {
    System.out.println(status);
}
```

## Предимства

Използването на enum:

- ограничава допустимите стойности;
- подобрява четимостта на програмния код;
- осигурява проверка на типовете още при компилация;
- намалява вероятността от грешки.

## Особености

Изброените типове могат да съдържат:

- полета;
- конструктори;
- методи.

Това ги прави специален вид класове, а не просто списък от константи.

Използването на enum е препоръчително винаги, когато е необходимо дадена променлива да приема една от предварително известен и ограничен набор стойности (например статус на поръчка, ден от седмицата, сезон, вид плащане и др.).

## Записи (record)

Записът (record) е специален вид клас, предназначен за съхраняване на данни.

Той позволява компактно описание на неизменими обекти, без необходимост от ръчно дефиниране на конструктори, getter методи и други стандартни елементи.

## Декларация

```java
record Student(String name,
               int facultyNumber) {

}
```

## Създаване на обект

```java
Student student =
        new Student("Ivan", 12345);
```

## Достъп до стойностите

За всеки компонент автоматично се създава метод със същото име.

```java
System.out.println(student.name());

System.out.println(student.facultyNumber());
```

## Автоматично генерирани елементи

Компилаторът автоматично създава:

- конструктор;
- getter-подобни методи;
- equals();
- hashCode();
- toString().

## Предимства

Използването на record:

- намалява количеството шаблонен код;
- подобрява четимостта;
- улеснява създаването на класове, които служат единствено за пренасяне на данни.

## Ограничения

- Компонентите на record са неизменими.
- След създаване на обекта техните стойности не могат да бъдат променяни.

Пример:

```java
record Point(int x, int y) {

}

public class Application {

    public static void main(String[] args) {
        Point point = new Point(10, 20);
        System.out.println(point);
    }
}
```
