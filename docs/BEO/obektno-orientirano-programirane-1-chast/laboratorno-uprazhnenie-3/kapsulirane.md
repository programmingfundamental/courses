# Капсулиране (Encapsulation)

**Капсулацията** е един от основните принципи на обектно-ориентираното програмиране. Тя представлява обединяване на данните и методите, които работят с тях, в рамките на един клас, като едновременно с това се ограничава директният достъп до вътрешното състояние на обекта.

Основната цел на капсулацията е да предпази обекта от неправилна или неконтролирана промяна на неговите данни.

**Реализиране на капсулацията**

Капсулацията се реализира чрез:

- ограничаване на директния достъп до полетата на класа;
- предоставяне на контролиран достъп чрез публични методи;
- извършване на проверки (валидация) при промяна на стойностите.

Пример:

```java
class Student {

    private String name;
    private int facultyNumber;

    public String getName() {
        return name;
    }

    public void setFacultyNumber(int facultyNumber) {
        if (facultyNumber > 0) {
            this.facultyNumber = facultyNumber;
        }
    }
}
```

В примера директният достъп до полетата е ограничен. Четенето и промяната на стойностите се извършват чрез методите на класа.

**Предимства**

Използването на капсулация предоставя следните предимства:

- защита на вътрешното състояние на обекта;
- контрол върху достъпа до данните;
- възможност за проверка на стойностите преди тяхното използване;
- по-лесна поддръжка и разширяване на програмния код.

## Примери за капсулиране

### 1. Модифицируеми обекти (Mutable Objects)
При тях състоянието може да се променя след създаването на обекта чрез използване на **setter методи**.

```java
class Person {
    private String name;
    private int age;

    // Гетър за името
    public String getName() {
        return name;
    }

    // Сетър за името - позволява промяна
    public void setName(String name) {
        this.name = name;
    }

    // Сетър за годините с вградена валидация
    public void setAge(int age) {
        if (age > 0) {
            this.age = age;
        }
    }
}
```

### 2. Немодифицируеми обекти (Immutable Objects)
При тях състоянието се задава веднъж при създаването (чрез конструктор) и не може да бъде променено по-късно. Това се постига чрез:

- Деклариране на полетата като `private final`.
- Предоставяне само на **getter методи** (без сетери).

```java
class ImmutablePerson {
    private final String name; // 'final' гарантира, че стойността няма да се промени

    public ImmutablePerson(String name) {
        this.name = name;
    }

    public String getName() {
        return name;
    }
}
```

---

## Предимства на капсулирането

- **Контрол и валидация:** Чрез setter методите е възможен контрол какви стойности се присвояват на декларираните променливи (например да се предотврати задаването на отрицателна възраст).
- **Гъвкавост:** Възможно е даден атрибут да се декларира „само за четене“ (readonly), което се постига с липса на setter метод за него.
- **Скриване на сложността:** Потребителят на класа не е необходимо да знае как точно се съхраняват данните вътрешно. Той използва само методите, предоставени от публичния интерфейс.
- **Лесна поддръжка:** При възникнала необходимост от промяна на името или типа на някоя вътрешна променлива, ще е необходима актуализация само на методите вътре в самия клас, без да това да се отрази на останалата част от програмния код, който го използва.
- **Сигурност:** Предпазва обекта от преминаване в невалидно състояние.
