# Нуждата от Wrapper Class

Въпреки че има примитивни типове, в java се налага използване на така наречените Wrapper (обгръщащи) класове заради допълнителните функционалности, които са реализирани в тях.

В Wrapper Class се включват методи като:

1. valueOf() - за конвертиране на примитивни типове
2. parseInt() - за конвертиране от String
3. toString() - за текстово представяне

Полезни свойства на Wrapper Class:

1. Преобразува примитивните типове данни в обекти. Обектите са необходими в случай, че се налага промяна на аргументи, предавани в метод - примитивните типове се предават по стойност.
2. Класовете в пакета java.util обработват само обекти и следователно Wrapper Class помагат и в този случай.
3. Структурите от данни (колекции) като ArrayList и Vector съхраняват само обекти (референтни типове), а не примитивни типове.
4. Необходим е обект за поддържане на синхронизация при многонишково програмиране

Една от основните важни характеристики, предоставени от Wrapper Class, е множеството налични методи. При стойност с плаваща запетая и необходимост от намиране на нейната цяла част, може да се използва специфичният за тази цел съществуващ метод:

Пример

```
import java.io.*;

class Application {

    public static void main(String[] args) {

        Float floatWrap = Float.valueOf(45.158f);
        int floatToInt = floatWrap.intValue();
        System.out.println(floatToInt);

        Integer five = Integer.valueOf("101", 2);
        System.out.println(five);
    }
}
```
