# Полиморфизъм и абстрактни класове


Полиморфизмът е принцип в обектно-ориентираното програмиране, при който една и съща операция може да има различна реализация според конкретния тип на обекта. В Java това най-често се проявява при наследяване и предефиниране на методи.

Абстрактният клас е клас, който не е предназначен за директно създаване на обекти. Той описва обща структура и общо поведение, което трябва да бъде наследено и довършено от конкретни класове.

## Предефиниране на метод

Предефиниране на метод се получава, когато клас наследник дефинира метод със същата сигнатура като метод от родителския клас.

```java
class Animal {

    public String sound() {
        return "Unknown sound";
    }
}

class Dog extends Animal {

    @Override
    public String sound() {
        return "Bark";
    }
}
```

Методът `sound()` съществува в `Animal`, но класът `Dog` предоставя собствена реализация. Анотацията `@Override` указва, че методът трябва да предефинира наследен метод.

## Полиморфна референция

Променлива от родителски тип може да сочи към обект от клас наследник.

```java
Animal animal = new Dog();

System.out.println(animal.sound());
```

Типът на променливата е `Animal`, но реалният обект е `Dog`. При извикване на `sound()` се изпълнява реализацията от `Dog`. Изборът на метода се извършва по време на изпълнение според реалния тип на обекта.

## Масив от родителски тип

Полиморфизмът позволява различни наследници да се обработват чрез общ родителски тип.

```java
Animal[] animals = new Animal[3];

animals[0] = new Dog();
animals[1] = new Cat();
animals[2] = new Bird();

for (Animal animal : animals) {
    System.out.println(animal.sound());
}
```

Всеки елемент от масива има тип `Animal`, но може да съдържа различен конкретен наследник. При обхождане се извиква подходящата реализация на `sound()`.

## Абстрактен клас

Абстрактен клас се декларира с ключовата дума `abstract`.

```java
abstract class Shape {

    private String color;

    public Shape(String color) {
        this.color = color;
    }

    public String getColor() {
        return color;
    }

    public abstract double calculateArea();
}
```

Класът `Shape` съдържа поле, конструктор, обикновен метод и абстрактен метод. Абстрактният метод `calculateArea()` няма тяло. Всеки конкретен наследник трябва да предостави реализация.

## Наследяване на абстрактен клас

```java
class Rectangle extends Shape {

    private double width;
    private double height;

    public Rectangle(String color, double width, double height) {
        super(color);
        this.width = width;
        this.height = height;
    }

    @Override
    public double calculateArea() {
        return width * height;
    }
}
```

Класът `Rectangle` наследява `Shape`, извиква конструктора на родителския клас чрез `super(color)` и реализира абстрактния метод `calculateArea()`.

## Правила за абстрактни класове

От абстрактен клас не може да се създава директен обект.

```java
// Shape shape = new Shape("red"); // не се компилира
```

Абстрактен клас може да съдържа полета, конструктори, обикновени методи и абстрактни методи. Ако един клас съдържа поне един абстрактен метод, самият клас трябва да бъде деклариран като `abstract`.

## Предефиниране и претоварване

Предефинирането не е същото като претоварване на методи. Претоварването е въведено при разглеждането на класове и
методи и се избира при компилация според параметрите на метода.

Предефинирането е свързано с наследяване. При него клас наследник предоставя нова реализация на метод със същата
сигнатура като метод от родителския клас. Изборът на предефиниран метод се извършва по време на изпълнение според
реалния тип на обекта.
