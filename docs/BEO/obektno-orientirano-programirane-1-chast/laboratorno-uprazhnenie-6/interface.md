# Интерфейс (Interface)

Интерфейсът представлява механизъм за описване на поведение, което може да бъде реализирано от един или повече класове. Той определя какво трябва да може да извършва даден обект, без да определя как ще бъде реализирано това поведение.

Интерфейсите се използват за дефиниране на общ договор (contract), който класовете, реализиращи интерфейса, са длъжни да спазват.

Декларацията на интерфейс се извършва чрез ключовата дума interface.

## Декларация на интерфейс

Най-простият интерфейс съдържа декларации на методи без тяхната реализация.

```java
public interface Printable {

    void print();

}
```

## Имплементиране на интерфейс

Класът реализира интерфейс чрез ключовата дума implements.

Всички абстрактни методи на интерфейса трябва да бъдат реализирани. Ако това не бъде направено, класът също трябва да бъде деклариран като абстрактен.

```java
public interface Printable {

    void print();

}

public class Document implements Printable {

    @Override
    public void print() {
        System.out.println("Printing document...");
    }

}
```

## Какво може да съдържа един интерфейс?

След Java 8 интерфейсите могат да съдържат различни видове членове.

В интерфейс могат да бъдат декларирани:

- абстрактни методи;
- методи с реализация (default);
- статични методи (static);
- константи (public static final).

## Абстрактни методи

Абстрактният метод съдържа само декларация и трябва да бъде реализиран от всеки клас, който имплементира интерфейса.

```java
public interface Drawable {

    void draw();

}
```

## Методи с реализация (default)

Методите с модификатор *default* имат собствена реализация и не е задължително да бъдат предефинирани.

```java
public interface Printable {

    void print();

    default void preview() {
        System.out.println("Preview");
    }

}
```

## Статични методи

Интерфейсите могат да съдържат и статични методи. Те принадлежат на самия интерфейс и се извикват чрез неговото име.

```java
public interface Printable {

    static void printInfo() {
        System.out.println("Printable interface");
    }

}
```

Извикване:

```java
Printable.printInfo();
```

## Константи

Всички полета, декларирани в интерфейс, са автоматично:

- public;
- static;
- final.

```java
public interface Constants {

    int MAX_SIZE = 100;

}
```

Полетата в интерфейсите представляват константи (public static final). На практика обаче интерфейсите се използват основно за дефиниране на поведение, а не за съхраняване на константи. При необходимост константите обикновено се дефинират в отделен помощен клас или като елементи на enum, когато описват фиксиран набор от стойности.

## Наследяване на интерфейси

Подобно на класовете, интерфейсите също могат да наследяват други интерфейси.

При интерфейсите се използва ключовата дума extends. За разлика от класовете, един интерфейс може да наследява повече от един родителски интерфейс.

```java
public interface Printable {

    void print();

}

public interface Savable {

    void save();

}

public interface PrintableDocument extends Printable, Savable {

}
```

Всеки клас, който реализира PrintableDocument, трябва да реализира методите и от двата родителски интерфейса.

## Видове интерфейси

В практиката най-често се използват следните видове интерфейси:

## Обикновени интерфейси

Използват се за описание на общо поведение.

```java
public interface Drawable {

    void draw();

}
```

## Функционални интерфейси

Функционалният интерфейс съдържа точно един абстрактен метод. Той служи като основа за използване на ламбда изрази и функционално програмиране.

```java
@FunctionalInterface
public interface Calculator {

    int calculate(int a, int b);

}
```

## Маркиращи интерфейси

Маркиращият интерфейс не съдържа методи и полета.

Той служи единствено за обозначаване, че даден клас притежава определено свойство или трябва да бъде обработван по специален начин.

Примери за маркиращи интерфейси са:

- Serializable;
- Cloneable;
- Remote.

В съвременната Java за подобни цели често се използват анотации, но маркиращите интерфейси все още присъстват в стандартната библиотека.

## Особености при работа с интерфейси

При използването на интерфейси трябва да се имат предвид следните особености:

- не могат да бъдат създавани обекти от интерфейс;
- един клас може да реализира произволен брой интерфейси;
- интерфейсите могат да наследяват други интерфейси;
- всички полета в интерфейс представляват константи;
- класът, който реализира интерфейс, трябва да реализира всички негови абстрактни методи.
