## Абстрактни класове и интерфейси

Абстрактните класове и интерфейсите служат за реализиране на абстракция, но имат различно предназначение.

Абстрактният клас описва обща основа за група свързани класове, които споделят общо състояние и поведение. Интерфейсът описва общо поведение, което може да бъде реализирано от напълно различни класове. Интерфейсите от друга страна се използват за 100% абстракция.

| Абстрактен клас                                                                                   | Интерфейс                                                                                                     |
| ------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------- |
| Абстрактният клас се използва, когато класовете споделят общо състояние и част от поведението си. | Използва се, когато различни класове трябва да предоставят еднакво поведение, без да споделят общо състояние. |
| Може да съдържа състояния (полета)                                                                | Декларираните полета са константи.                                                                            |
| Един клас може да наследи само един абстрактен клас.                                              | Един клас може да реализира множество интерфейси.                                                             |
| Може да съдържа полета, конструктори, абстрактни и конкретни методи.                              | Може да съдържа константи, абстрактни, default и static методи.                                               |
| Подходящ е при наличие на обща реализация.                                                        | Подходящ е, когато различни класове трябва да реализират еднакво поведение.                                   |

## Кога да се използва абстрактен клас?

Абстрактният клас е подходящ, когато:

- класовете имат общи полета;
- съществува обща реализация на част от методите;
- изгражда се йерархия от свързани класове.

## Кога да се използва интерфейс?

Интерфейсът е подходящ, когато:

- различни класове трябва да предоставят едно и също поведение;
- един клас трябва да изпълнява няколко различни роли;
- не съществува естествена връзка "родител–наследник" между класовете.

Пример:

```java
public abstract class Shape {

    private String color;

    public Shape(String color) {
        this.color = color;
    }

    public String getColor() {
        return color;
    }

    public void printColor() {
        System.out.println("Color: " + color);
    }

    public abstract double area();

}

public interface Drawable {

    void draw();

}

public class Circle extends Shape implements Drawable {

    private double radius;

    public Circle(String color, double radius) {
        super(color);
        this.radius = radius;
    }

    @Override
    public double area() {
        return Math.PI * radius * radius;
    }

    @Override
    public void draw() {
        System.out.println("Drawing circle");
    }
}
```

Класът Shape описва какво представлява една геометрична фигура. Всички фигури имат цвят и могат да изчисляват лице, но формулата за изчисляване на лицето е различна за различните фигури. Поради тази причина методът area() е деклариран като абстрактен и всяка конкретна фигура трябва да предостави собствена реализация.

Интерфейсът Drawable не описва какъв е обектът, а само че той може да бъде изчертан. Всеки клас, който имплементира този интерфейс, е длъжен да реализира метода draw().

Класът Circle наследява абстрактния клас Shape, с което получава неговите полета и реализираните методи, и едновременно с това имплементира интерфейса Drawable. По този начин той наследява общите характеристики на геометричните фигури, реализира собствен алгоритъм за изчисляване на лицето и предоставя реализация на операцията за изчертаване.

В практиката абстрактните класове и интерфейсите често се използват заедно. Абстрактният клас предоставя общо състояние и споделена реализация, а интерфейсът описва допълнителна функционалност, която даден клас трябва да поддържа.
