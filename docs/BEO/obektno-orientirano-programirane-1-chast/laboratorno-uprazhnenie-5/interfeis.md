# Интерфейс

Интерфейсът изглежда като клас, но обикновено предимно дефинира поведение. По подразбиране, всички декларирани в интерфейс методи са абстрактни - те нямат тяло. Интерфейсът освен това може да съдържа декларации на полета, които по подразбиране са декларирани като public static final и се използват за константи.

Тъй като методите в интерфейсите нямат тяло, те трябва да бъдат имплементирани от клас, преди да се достъпват. Класът, който реализира интерфейса, трябва да имплементира всички методи на този интерфейс. В Java не е позволено наследяване на повече от един клас, но е напълно възможно имплементирането на повече от един интерфейс от един клас.

```
interface ExampleInterface {
   /* всички методи са публични и абстрактни по подразбиране
    * те нямат тяло
    */
   public void method1();
   public void method2();
}
class Demo implements ExampleInterface {
   /* класовете, който имплементират интерфейса,
    * трябва да реализират методите на интерфейса
    */
   public void method1()
   {
	System.out.println("implementation of method1");
   }
   public void method2()
   {
	System.out.println("implementation of method2");
   }
   public static void main(String arg[])
   {
	MyInterface obj = new Demo();
	obj.method1();
   }
}
class Main {
   public static void main(String arg[])
   {
	MyInterface obj = new Demo();
	obj.method1();
   }
}
```

### Интерфейс и наследяване

```
interface Inf1 {
   public void method1();
}
interface Inf2 extends Inf1 {
   public void method2();
}
public class Demo implements Inf2 {
   /* класът, имплементиращ интерфейс,
    * трябва да имплементира всички методи от интерфейса и
    * неговите родителски интерфейси
    */
    public void method1(){
	System.out.println("method1");
    }
    public void method2(){
	System.out.println("method2");
    }
    public static void main(String args[]){
	Inf2 obj = new Demo();
	obj.method2();
    }
}
```

### Маркиращи интерфейси

Празният интерфейс е известен като интерфейс за етикет или маркиращ интерфейс. Той не съдържа полета и методи, а служи за обозначаване ("маркиране") на принадлежност на даден клас към определена категория. Това може да предполага притежанието на специално свойство или необходимостта от специална обработка. В съвременната Java за подобни цели често се използват така наречените анотации, но подобни маркиращи интерфейси все още съществуват и се срещат в стандартни библиотеки. Примери за такива интерфейси са Serializable, EventListener, Remote и други.

### Важно

1. Не е възможно инстанциране на интерфейс в java. Това означава, че е невъзможно създаването на обект от интерфейс;
2. Интерфейсът осигурява пълна абстракция, тъй като никой от неговите методи няма тяло;
3. **implements** е ключовата дума, която се използва от класовете за имплементиране на интерфейс.
4. Класът, който имплементира даден интерфейс, трябва да имплементира всички методи на този интерфейс, в противен случай класът трябва да бъде абстрактен;
5. Интерфейсът не може да бъде деклариран като private или protected;
6. Всички методи на интерфейса по подразбиране са абстрактни и публични (не е необходимо указването на модификатор за достъп); 
7. Полетата, декларирани в интерфейса, са public static final по подразбиране.
