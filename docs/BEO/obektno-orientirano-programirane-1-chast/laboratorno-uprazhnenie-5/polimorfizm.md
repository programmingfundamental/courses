# Полиморфизъм

## Претоварване (overloading)

Претоварването (overloading) представлява възможност в един клас да съществуват няколко метода или конструктори с едно и също име, но с различен списък от параметри.

Компилаторът определя кой метод или конструктор да бъде извикан въз основа на броя, типа и последователността на подадените аргументи.

## Претоварване на методи

За да бъдат претоварени два метода, те трябва:

- да имат еднакво име;
- да приемат различен брой параметри или параметри от различен тип.

Промяната единствено на типа на връщаната стойност не е достатъчна за претоварване.

```java
public class Calculator {

    public int sum(int a, int b) {
        return a + b;
    }

    public int sum(int a, int b, int c) {
        return a + b + c;
    }

    public double sum(double a, double b) {
        return a + b;
    }
}
```

Компилаторът избира подходящия метод според аргументите, подадени при извикването му.

## Претоварване на конструктори

По същия начин могат да бъдат претоварвани и конструкторите.

Това позволява създаването на обекти по различни начини в зависимост от наличната информация.

```java
public class Student {

    private String name;
    private int facultyNumber;

    public Student() {
        name = "Unknown";
        facultyNumber = 0;
    }

    public Student(String name) {
        this.name = name;
        facultyNumber = 0;
    }

    public Student(String name, int facultyNumber) {
        this.name = name;
        this.facultyNumber = facultyNumber;
    }
}
```

В примера класът предоставя три различни начина за създаване на обект.

## Предефиниране на методи (method overriding)

Предефинирането (overriding) позволява клас наследник да предостави собствена реализация на метод, наследен от родителския клас.

При извикване на метода се изпълнява реализацията на действителния тип на обекта.

## Изисквания за презаписване на метод

За да бъде един метод предефиниран:

- трябва да има същото име;
- трябва да приема същия брой параметри;
- параметрите трябва да бъдат от същите типове;
- трябва да връща резултат от същия тип (или съвместим тип).

Добра практика е използването на анотацията @Override, която позволява на компилатора да провери дали методът е предефиниран коректно.

```java
public class Shape {

    public double area() {
        return 0;
    }
}

public class Circle extends Shape {

    private double radius;

    public Circle(double radius) {
        this.radius = radius;
    }

    @Override
    public double area() {
        return Math.PI * radius * radius;
    }
}
```

## Полиморфизъм (Polymorphism)
Думата полиморфизъм произлиза от гръцки език и означава „много форми“.

В програмирането полиморфизмът представлява възможността едно име или една референция да бъдат използвани по различен начин в зависимост от конкретната ситуация.

В Java се разграничават два основни вида полиморфизъм:

- полиморфизъм по време на компилация (compile-time polymorphism);
- полиморфизъм по време на изпълнение (runtime polymorphism).

## Полиморфизъм по време на компилация

Полиморфизмът по време на компилация се реализира чрез **претоварване на методи и конструктори (overloading)**.

Компилаторът избира кой метод да бъде извикан въз основа на броя, типа и последователността на подадените аргументи.

## Полиморфизъм по време на изпълнение

Полиморфизмът по време на изпълнение се реализира чрез **предефиниране на методи (overriding) и наследяване**.

При този вид полиморфизъм изборът на конкретната реализация не се извършва от компилатора, а по време на изпълнение на програмата в зависимост от реалния тип на обекта.

## Пример за използване на полиморфизъм

```java
Shape shape = new Circle(5);

System.out.println(shape.area());
```

Променливата *shape* е декларирана като Shape, но сочи към обект от тип Circle. При извикване на метода *area()* се изпълнява реализацията на класа Circle.
