# Абстрактен клас

Абстрактният клас представлява специален клас, който служи като основа за създаване на други класове. Абстрактните класове се използват, когато няколко класа имат общи характеристики и поведение, но не съществува смислена реализация на всички техни методи в родителския клас. В този случай родителският клас описва общата структура, а конкретната реализация се предоставя от класовете наследници.

Той може да съдържа както реализирани методи, така и абстрактни методи – методи без реализация.

Абстрактният клас се декларира чрез ключовата дума abstract.

*Кога се използва абстрактен клас?*

Абстрактният клас е подходящ, когато:
* няколко класа споделят общи полета и методи;
* част от поведението е еднакво за всички наследници;
* друга част трябва задължително да бъде реализирана по различен начин във всеки наследник.


*Какво може да съдържа един абстрактен клас?*

Абстрактният клас може да съдържа:
* полета;
* конструктори;
* конкретни методи;
* абстрактни методи.

Това позволява общата функционалност да бъде реализирана само веднъж в родителския клас, а специфичното поведение да бъде оставено на наследниците.

```java
//абстрактен клас
public abstract class Shape {

    // attributes, constructor, methods

    public abstract double area();

}

//Наследник на класа Shape
public class Circle extends Shape {

    private double radius;

   public Circle(double radius) {
        this.radius = radius;
    }

    @Override
    public double area() {
        return Math.PI * radius * radius;
    }
}
```
Използване:
```java
Shape shape = new Circle(5);

System.out.println(shape.area());
```

### Правила

При работа с абстрактни класове трябва да се имат предвид следните особености:
* не могат да бъдат създавани обекти от абстрактен клас;
* абстрактният клас може да съдържа както абстрактни, така и конкретни методи;
* ако клас наследник не реализира всички абстрактни методи, той също трябва да бъде деклариран като абстрактен.

### Предимства

Използването на абстрактни класове:
* позволява извеждането на обща функционалност в родителски клас;
* намалява дублирането на код;
* гарантира, че наследниците ще реализират определени методи;
* улеснява използването на полиморфизъм.

**Важно:** Абстрактният клас не съществува самостоятелно. Той описва общите характеристики на група свързани класове и обикновено се използва като основа за изграждане на йерархия от наследници.  

**Обобщение**

| Понятие         | Предназначение                                                                     |
|-----------------|------------------------------------------------------------------------------------|
| Претоварване    | Един метод (или конструктор) с няколко различни параметрични списъка.              |
| Предефиниране   | Нова реализация на наследен метод.                                                 |
| Полиморфизъм    | Работа с различни обекти чрез общ родителски тип.                                  |
| Абстрактен клас | Базов клас, който не може да бъде инстанциран и може да съдържа абстрактни методи. |
