#### StringBuilder

Поради неизменимостта на String, многократната промяна на един символен низ води до създаване на множество временни обекти.

Класът *StringBuilder* решава този проблем чрез използване на **модифицируем буфер**, чието съдържание може да бъде променяно без създаване на нови обекти.

```java
StringBuilder sb = new StringBuilder(15);
sb.append("Hello, World! ");
sb.append("It’s a great day!");
```

**Най-често използвани методи**

| Метод         | Предназначение                    |
|---------------|-----------------------------------|
| append()      | добавяне                          |
| insert()      | вмъкване                          |
| delete()      | изтриване                         |
| replace()     | замяна                            |
| charAt()      | символ по индекс                  |
| length()      | дължина                           |
| capacity()    | размер на буфера                  |
| toString()    | преобразуване към String          |

#### StringBuffer

*StringBuffer* предоставя почти същата функционалност като StringBuilder.

Основната разлика е, че методите му са **синхронизирани**, което позволява безопасното му използване от няколко нишки едновременно.

Поради синхронизацията StringBuffer обикновено е по-бавен от StringBuilder.

#### Сравнение между String, StringBuilder и StringBuffer

| String                               | StringBuilder               | StringBuffer                                     |
|--------------------------------------|-----------------------------|--------------------------------------------------|
| immutable                            | mutable                     | mutable                                          |
| При промяна се създава нов обект     | Променя текущия буфер       | Променя текущия буфер                            |
| Подходящ за неизменяем текст         | Подходящ при чести промени  | Подходящи пи чести промени и многонишкова среда  |
| Thread-safe поради неизменимостта си | Не                          | Да                                               |
| Най-често използван                  | Най-бърз                    | По-бавен заради синхронизацията                  |

#### ASCII и Unicode

Компютрите съхраняват символите като числови кодове.

Първоначално за тази цел се е използвала таблицата ASCII, която съдържа 128 символа (или 256 в разширения вариант). Тя е достатъчна за английската азбука, цифрите и основните специални символи, но не позволява представянето на повечето национални азбуки.

Поради тази причина Java използва Unicode, който предоставя единна кодова система за символите на почти всички съвременни езици. Това позволява в една и съща програма едновременно да се използват текстове на латиница, кирилица, гръцки, арабски, китайски и много други писмености.
